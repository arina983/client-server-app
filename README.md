# Backend-сервер на C++ для приёма данных с Android

## Описание проекта

Проект представляет собой **backend-сервер на C++**, предназначенный для приёма данных о местоположении смартфона и информации о мобильных сетях (Cell Info) с Android-приложения. Сервер получает информацию о местоположении (GPS) и параметрах сотовой связи (Cell Info), включая данные о нескольких базовых станциях одновременно.
Сервер принимает данные по протоколу ZeroMQ, обрабатывает их и сохраняет в реляционную базу данных PostgreSQL с поддержкой истории всех видимых базовых станций (PCI).


### Основные возможности

- Приём данных о местоположении (GPS) и информации о сотовых сетях (LTE, GSM, NR/5G)
- Сохранение всех полученных пакетов в файл `visual.json`
- Многопоточная архитектура с безопасным доступом к данным (`std::mutex`)
- Автоматическое обновление интерфейса при поступлении новых данных
- Полная поддержка параметров LTE: **RSRP, RSRQ, RSSI, SINR** для каждой PCI
- PostgreSQL: Использование связи «Один ко многим» (1:N) для хранения истории всех соседних сот.
- Поддержка нескольких сот одновременно (разные цвета на графиках + легенда с PCI)

## Технологический стек

- **Язык:** C++
- **Сетевой обмен:** ZeroMQ
- **Графический интерфейс:** Dear ImGui + ImPlot
- **Многопоточность:** `std::thread`, `std::mutex`

## База данных

device_data: Основная таблица для GPS-координат и параметров активной соты.

cells_history: Связанная таблица для хранения параметров всех обнаруженных соседних сот (PCI). Целостность данных обеспечивается через Foreign Key с правилом ON DELETE CASCADE.

## Архитектура

- **Поток 1 (ZeroMQ-сервер)** — принимает данные от Android-приложения, парсит их и обновляет общую структуру `location`.
- **Поток 2 (GUI)** — отображает актуальные данные и графики в реальном времени.
- Связь между потоками осуществляется через общую структуру `location` + `std::mutex`.

## Как запустить

### 1. Сборка проекта

Должны быть установлены следующие зависимости:
- PostgreSQL
- ZeroMQ
- SDL2
- CMake
- Dear ImGui + ImPlot (включены в проект)

### 2. Подготовка базы данных

Создайте базу данных:

```sql
CREATE DATABASE mobile_data;


Сборка проекта:

```bash
mkdir build && cd build
cmake --build .
make
```

### 2. Запуск
```bash
./main
```

## Особенности реализации

- Автоматическое переподключение Android-клиента реализуется на стороне Android (try-catch + повторные попытки подключения).
- Сервер продолжает работать даже при временном отключении клиента.
- Все данные сохраняются в `visual.json` (append-режим).
