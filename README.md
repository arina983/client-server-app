# Backend-сервер на C++ для приёма данных с Android

## Описание проекта

Данный проект представляет собой **backend-сервер на C++**, предназначенный для приёма данных о местоположении смартфона и информации о мобильных сетях (Cell Info) с Android-приложения.

Сервер принимает данные по протоколу **ZeroMQ**, сохраняет их в JSON-файл, обновляет общую структуру данных и отображает информацию в удобном графическом интерфейсе на базе **Dear ImGui + ImPlot**.

### Основные возможности

- Приём данных о местоположении (GPS) и информации о сотовых сетях (LTE, GSM, NR/5G)
- Сохранение всех полученных пакетов в файл `visual.json`
- Многопоточная архитектура (отдельные потоки для ZeroMQ и GUI)
- Автоматическое обновление интерфейса при поступлении новых данных
- Красивые графики истории сигнала: **RSRP**, **RSRQ**, **RSSI**
- Поддержка парсинга сложных строк с данными CellInfo
- Потокобезопасный доступ к данным через `std::mutex`

## Технологический стек

- **Язык:** C++
- **Сетевой обмен:** ZeroMQ
- **Графический интерфейс:** Dear ImGui + ImPlot
- **Бэкенд рендеринга:** SDL2 + OpenGL
- **Многопоточность:** `std::thread`, `std::mutex`

## Архитектура

- **Поток 1 (ZeroMQ-сервер)** — принимает данные от Android-приложения, парсит их и обновляет общую структуру `location`.
- **Поток 2 (GUI)** — отображает актуальные данные и графики в реальном времени.
- Связь между потоками осуществляется через общую структуру `location` + `std::mutex`.

## Как запустить

### 1. Сборка проекта

Убедитесь, что у вас установлены следующие библиотеки:

- ZeroMQ
- SDL2
- Dear ImGui + ImPlot (включены в проект)

Затем соберите проект:

```bash
mkdir build && cd build
cmake --build .
make
```

### 2. Запуск
```bash
./main
```
Сервер запустится на порту **5551** (`tcp://0.0.0.0:5551`).

## Формат данных от Android

Android-приложение должно отправлять строку в следующем формате:

```
lon,lat,alt,accuracy,time | CellInfoString | RX=123456 TX=789012
```
## Функционал GUI

В графическом интерфейсе отображаются:

- **Location** — широта, долгота, высота, точность, время
- **Cell Information** с цветовой подсветкой по типу сети:
  - Зелёный — LTE (4G)
  - Жёлтый — GSM
  - Красный — NR (5G)
- **Network Traffic** (RX/TX в байтах и МБ)
- **Графики сигналов**:
  - RSRP (dBm)
  - RSRQ (dB)
  - RSSI (dBm)

## Важные особенности реализации

- Автоматическое переподключение Android-клиента реализуется на стороне Android (try-catch + повторные попытки подключения).
- Сервер продолжает работать даже при временном отключении клиента.
- Все данные сохраняются в `visual.json` (append-режим).
